# Планы
Разделим реализацию на четкие, последовательные этапы. Это позволит системно подойти к разработке и легко отслеживать прогресс.

### **Этап 0: Подготовка и настройка проекта**

1.  **Создание проекта:**
    *   Создать новое WPF-приложение в Visual Studio с именем `AviationDB.WPF`. ✅
    *   Удалить автоматически созданные `MainWindow.xaml` и `MainWindow.xaml.cs` (мы создадим свои). ✅

2.  **Установка NuGet пакетов:**
    *   Выполнить в Консоли диспетчера пакетов команды:
        *   `Install-Package Microsoft.EntityFrameworkCore`✅
        *   `Install-Package Npgsql.EntityFrameworkCore.PostgreSQL`✅
        *   `Install-Package Microsoft.Extensions.DependencyInjection` (для внедрения зависимостей)✅

3.  **Создание структуры папок:**
    *   В проекте создать папки: `Core`, `Infrastructure`, `ViewModels`, `Views`, `Services`, `docs`.✅

---

### **Этап 1: Модели и Контекст данных (Слой Core и Infrastructure)**

1.  **Определение моделей (Core):**
    *   В папке `Core` создать классы-модели (Entity), которые отражают структуру таблиц в вашей базе данных PostgreSQL.✅
    *   *Пример:* `Aircraft.cs`, `Pilot.cs`, `Flight.cs`.
    *   Продумать и реализовать связи между моделями (навигационные свойства).✅

2.  **Создание DbContext (Infrastructure):**
    *   В папке `Infrastructure` создать класс `AviationDbContext`, наследующий от `DbContext`.✅
    *   Добавить в него `DbSet<T>` для каждой модели.✅
    *   Переопределить метод `OnConfiguring` для временного указания строки подключения (на этапе разработки).✅
    *   Переопределить метод `OnModelCreating` при необходимости для настройки связей, ключей, индексов.✅

3.  **Миграции базы данных:**
    *   Выполнить команду добавления миграции (например, `Add-Migration InitialCreate`).❌
    *   Выполнить команду обновления базы данных (`Update-Database`), чтобы создать таблицы в PostgreSQL.❌

---

### **Этап 2: Репозитории (Слой Infrastructure)**

1.  **Определение интерфейсов репозиториев (Core):**
    *   В папке `Core` создать интерфейсы для каждого репозитория (по одному на модель).✅
    *   *Пример:* `IAircraftRepository.cs`.
    *   Интерфейсы должны объявлять стандартные CRUD-методы: `GetAll`, `GetById`, `Add`, `Update`, `Delete`.✅

2.  **Реализация репозиториев (Infrastructure):**
    *   В папке `Infrastructure` создать классы, реализующие соответствующие интерфейсы (например, `AircraftRepository : IAircraftRepository`).✅
    *   Внутри репозиториев использовать `AviationDbContext` для выполнения операций с базой данных.✅
    *   Реализовать все методы, описанные в интерфейсах.✅

---

### **Этап 3: Настройка Внедрения Зависимостей (DI)**

1.  **Создание контейнера сервисов:**
    *   В файле `App.xaml.cs` создать и настроить `ServiceCollection` (из `Microsoft.Extensions.DependencyInjection`).
    *   Зарегистрировать `AviationDbContext` (желательно в режиме Scoped).
    *   Зарегистрировать репозитории, связав интерфейсы с их реализациями.
    *   Зарегистрировать все ViewModels.

2.  **Создание провайдера сервисов:**
    *   Построить `ServiceProvider` и сохранить его в свойство `App.Current` (например, в `App.Current.Resources` или в статическом свойстве), 
        чтобы к нему был доступ из всего приложения.

---

### **Этап 4: Реализация ViewModels (Слой ViewModels)**

1.  **Базовые ViewModels:**
    *   Создать базовый класс `ViewModelBase`, который реализует интерфейс `INotifyPropertyChanged` для уведомления об изменении свойств.

2.  **Главная ViewModel:**
    *   Создать `MainViewModel`. Она будет содержать другие ViewModels и управлять текущим отображаемым представлением (например, с помощью свойства `CurrentViewModel`).

3.  **ViewModels для сущностей:**
    *   Для каждой модели (Самолет, Пилот и т.д.) создать свою ViewModel (например, `AircraftViewModel`).
    *   Эти ViewModels будут:
        *   Содержать коллекции объектов (например, `ObservableCollection<Aircraft> Aircrafts`).
        *   Содержать выбранный объект (например, `Aircraft SelectedAircraft`).
        *   Содержать команды (ICommand) для выполнения операций CRUD (Загрузить, Добавить, Сохранить, Удалить).
        *   Через DI получать в конструкторе соответствующий репозиторий для работы с данными.

---

### **Этап 5: Создание Представлений (Слой Views)**

1.  **Главное окно (MainWindow):**
    *   Создать `MainWindow.xaml` в корне проекта.
    *   В разметке определить области для отображения других представлений (например, с помощью `ContentControl`, привязанного к `CurrentViewModel` из `MainViewModel`).
    *   Установить `DataContext` окна на `MainViewModel` через DI.

2.  **UserControl'ы для сущностей:**
    *   Для каждой ViewModel в папке `Views` создать соответствующий `UserControl` (например, `AircraftView.xaml`).
    *   В этих представлениях:
        *   Привязать `DataContext` к соответствующей ViewModel.
        *   Реализовать интерфейс для отображения списка объектов (например, с помощью `DataGrid`).
        *   Реализовать элементы управления (кнопки) для вызова CRUD-операций, привязав их к командам из ViewModel.
        *   Добавить поля ввода (TextBox) для создания/редактирования объектов, привязанные к свойствам выбранного элемента.

---

### **Этап 6: Связывание и Финальная настройка**

1.  **Навигация:**
    *   Реализовать в `MainViewModel` логику переключения между дочерними ViewModels (например, с помощью команд "Показать самолеты", "Показать пилотов").

2.  **Обработка ошибок:**
    *   Добавить базовую обработку исключений в репозиториях и ViewModels (например, с помощью `try-catch` и вывода сообщений в MessageBox).

3.  **Тестирование:**
    *   Запустить приложение.
    *   Протестировать все CRUD-операции для каждой таблицы: создание, чтение, обновление, удаление записей.
    *   Проверить, что изменения корректно сохраняются в базе данных PostgreSQL.

---

### **Этап 7: Документация (docs)**

1.  **Создание документации:**
    *   В папке `docs` создать файл `README.md`.
    *   Описать структуру проекта, используемые технологии.
    *   Указать шаги для настройки строки подключения к своей базе данных PostgreSQL.
    *   Привести примеры основных операций.

После завершения всех этапов у вас будет готовое WPF-приложение с архитектурой MVVM, выполняющее все требуемые функции.
